Section 1 — Before You Start (Admin)

Complete all items below before calling your friend. If any are missing, fix them first.

  • ngrok is running — terminal shows Forwarding https://unshackle-idealism-cardstock.ngrok-free.dev → http://localhost:3000. If it's not, run ngrok http 3000 in a new terminal.
  • Dev server is running — npm run dev is active in a separate terminal. Visit http://localhost:3000 to confirm.
  • NEXT_PUBLIC_APP_URL is set — .env.local has NEXT_PUBLIC_APP_URL=https://unshackle-idealism-cardstock.ngrok-free.dev. If you restarted the dev server since last session, verify this is still there.
  • Telegram bot is connected — open http://localhost:3000/dashboard/settings and confirm the Telegram card shows "Connected".
  • Admin Telegram is linked — Settings → Admin notifications card shows "Linked". If not, generate a fresh linking code and send /linkadmin <code> to the bot.
  • Google Calendar is connected — Settings → Google Calendar shows "Connected". Required for Tests 3 and 4.
  • 3 test classes exist — open http://localhost:3000/dashboard/classes and confirm: P5 Mathematics @ Tampines Hub (seats available), Sec 3 A-Math @ Bishan Junction 8 (FULL), JC1 H2 Mathematics @ Jurong East JEM (seats available).
  • Send your friend the bot link — share https://t.me/<YourBotUsername> with your friend and ask her to open it and tap Start. She should see a greeting (or blank; doesn't matter). Have her ready to chat.

Full booking flow: bot collects details, confirms, writes booking row, creates Google Calendar event, sends admin ping.

    • Your name: Mrs Tan
    • Phone: +6591234567
    • Email: (use your real email so you get the invite)

  4. Note the confirmation message — it should include date, time, and Tampines Hub address.
Admin (You)
  1. Watch your Telegram for a 🎉 New trial booking ping. Should arrive within seconds of your friend confirming.
  2. Check the ping contains: Sarah, P5 Mathematics, Tampines Hub, next Wednesday 5–6:30pm.
  3. Open /dashboard/bookings — new row should appear with "Cal. synced" badge and centre "Tampines Hub".
  4. Check your Google Calendar — event "Trial: Sarah — Mathematics (P5)" on next Wednesday 5pm SGT.
  5. Check your email — Google Calendar invite should have arrived from the calendar.
✓ Pass if: (1) Admin gets Telegram 🎉 ping. (2) Booking row in dashboard with synced badge. (3) Calendar event on correct date/time with Tampines Hub as venue. (4) Parent's Telegram receives the calendar invite line "I've also sent a calendar invite to…".

TEST 5
Escalation + Admin Takeover — Parent complains, admin steps in

Full escalation loop: parent is upset → bot flags it → admin gets pinged → admin takes over → admin replies directly to parent via dashboard → parent receives it in Telegram → admin resolves.

⚠ Before starting: Make sure you're logged into the dashboard at http://localhost:3000/dashboard/conversations and can see it during the test.
Parent (Your Friend)
  1. Send: I'm very unhappy — the teacher was late last week and nobody apologised
  2. Bot should reply with something like "I understand your concern, let me connect you with our team…"
  3. Wait for a direct message from the admin (this comes via the same bot chat).
  4. Reply to whatever the admin sends.
Admin (You)
  1. Watch your Telegram — a 🚨 Parent escalation ping should arrive with the parent's message and a link to the conversation.
  2. Tap the link — opens /dashboard/conversations/<id>.
  3. Click "Take over" — amber banner appears: "You've taken over this conversation".
  4. Type in the reply box: Hi Mrs Tan, this is the centre manager. I'm so sorry to hear this — let me look into it right away. Can I call you back in the next 30 minutes?
  5. Click Send.
  6. Watch for your friend's reply to appear live in the conversation viewer (without refreshing).
  7. Once resolved, click "Mark resolved".
✓ Pass if: (1) Admin gets 🚨 ping. (2) Conversation viewer shows "Take over" button. (3) After takeover, bot stops auto-replying. (4) Admin's typed reply appears in your friend's Telegram chat (sent by the bot but with your words). (5) Friend's next message appears in the dashboard viewer in real time (Supabase Realtime). (6) After "Mark resolved", conversation status in the list changes to "closed".
Note on real-time: The friend's reply should appear in the dashboard without you refreshing the page. If it doesn't appear, wait 5 seconds — Supabase Realtime can have a short connection warmup.
TEST 6
PDPA Opt-Out — Parent sends STOP

Verifies the opt-out flow closes the conversation and stops the bot from responding.

Parent (Your Friend)
  1. Send: STOP
  2. Try sending another message after: Actually wait, I have a question
Admin (You)
  1. Check /dashboard/conversations — the conversation should be marked "closed".
  2. After the second message, the bot should reply with "This chat has been closed…" and not engage further.
✓ Pass if: STOP gets confirmation reply. Conversation marked closed in dashboard. Follow-up message gets a static "closed" response, not a bot answer.

Section 4 — If Something Goes Wrong

SymptomLikely causeFix
Friend messages bot, no reply ngrok tunnel expired or dev server crashed Check ngrok terminal — restart if needed. Check dev server terminal. Then re-send the message.
Bot replies but takes 30+ seconds OpenAI API cold start or slow connection Normal on first message. Should be ~3–5s after warm-up.
🎉 admin ping doesn't arrive Admin Telegram not linked or bot token expired Settings → Admin notifications → re-link via /linkadmin.
Calendar event not created Google Calendar disconnected or token expired Settings → Google Calendar → disconnect and reconnect.
Admin reply doesn't reach friend Telegram bot token or channel_user_id mismatch Check dev server terminal for [telegram webhook] send failed. Reconnect the bot in Settings.
Realtime doesn't update dashboard Migration 009 not applied or Realtime not enabled Run migration 009 in Supabase. Then Supabase Studio → Database → Replication → confirm messages and conversations are in the publication.
Deep link from web doesn't show bot telegram_bot_username not populated Settings → Telegram → disconnect → reconnect. This backfills the username.
